# Lições Aprendidas — The Editor

*Este arquivo é atualizado pelo mecanismo de refinamento do The Editor.*
*Conforme episódios forem publicados e analisados, as lições são registradas aqui.*
*O Editor consulta este arquivo antes de produzir episódios do mesmo pilar.*

---

## Template de entrada

Cada lição usa o seguinte formato. Preencher todos os campos — lição incompleta não é lição.

```
### Lição [N] — [Título descritivo] — Pilar: [nome] — [Mês/Ano]

**Episódio:** Ep.[número] — [Título do episódio]
**O que aconteceu:** [Contexto objetivo — o que foi produzido, o que aconteceu na publicação ou no feedback]
**O que funcionou / não funcionou:** [Observação concreta — métricas de abertura/CTR, feedback do Job, comparação com episódios anteriores, ou análise editorial]
**A regra derivada:** [Instrução clara para o Editor aplicar nos próximos episódios deste pilar — começa com verbo no imperativo]
**Incorporado ao pilar:** [Sim / Não — se Sim, indicar o arquivo atualizado e o que mudou]
```

---

## Como usar este arquivo

Antes de produzir um episódio, o Editor verifica se há lições registradas para o pilar em questão. A **regra derivada** é o campo que importa — é o que muda na produção. Lições sem regra derivada não têm valor operacional.

Lições se acumulam por pilar. Quando um pilar tiver 5+ lições, o Editor revisa se alguma pode ser incorporada permanentemente à referência do pilar (`references/pilar-[nome].md`) e arquivada aqui como "incorporada".
